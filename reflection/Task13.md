# Рефлексия

В моём решении для Java отсутствует второй вариант скрытия методов. Частично, он в Java поддерживается, но при
этом остаётся возможность обратиться к методу класса-родителя из того же пакета, так как он помечен аннотацией 
protected. Ну и в предложенном решении есть маленькая опечатка: в классе B в методе getB() вместо this должно быть
super, а то рекурсивный вызов получается
